Key changes
- Potentials can connect to pins: when drawing a potential, the cursor snaps to a nearby pin (tolerance ~0.8mm). A small green circle highlights the pin. - Added language selector in Settings and persisted the selected language in localStorage.

Latest update: built-in component library + development export flow
- Refactored the component system into two sources:
  - App Library: built-in components loaded from JSON files in src/library/components/*.json
  - Project Components: user-saved components stored inside the project JSON file
- Added src/library/index.ts to eagerly load built-in component JSON files through import.meta.glob and expose them as read-only library items.
- Added src/utils/components.ts with reusable helpers for:
  - creating a component definition from selected shapes
  - preserving grid offsets for placement
  - slug generation for stable file names/ids
  - app-library file schema creation
  - runtime validation of component/library payloads
  - placement-key parsing for app vs project component placement
- Components panel was redesigned:
  - separate sections for App Library and Project Components
  - search/filter input
  - source/category badges
  - inline export and delete actions for project components
  - project save action now explicitly means save to project
- Replaced the Vite development-only write-to-disk export with a browser download flow.
- Built-in library export flow:
  1. draw shapes on canvas
  2. save as project component
  3. click Download JSON on that project component
  4. choose file name/category
  5. browser downloads a versioned component JSON file
  6. later place that file into src/library/components and rebuild the app
- Project file schema remains strict/current-only and still stores only project components, not the built-in app library.
- Added src/vite-env.d.ts so import.meta.glob is typed correctly.
- npm run build passes after the refactor.
