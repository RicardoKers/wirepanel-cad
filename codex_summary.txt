Current state
- The app is a browser-based 2D CAD for industrial electrical drawings built with React, TypeScript and Vite.
- `npm run build` passes.
- The project is connected to GitHub at `https://github.com/RicardoKers/basic2dcad`.
- GitHub Pages deployment is configured for the project site at `https://ricardokers.github.io/basic2dcad/`.
- Code structure, comments and variable names remain in English.

Environment and repo
- Node and npm were revalidated after machine format and reinstall.
- Dependencies install correctly.
- `.gitignore` was improved and `tsconfig.tsbuildinfo` is no longer tracked.
- The repository is configured for automatic deployment on push to `main` through `.github/workflows/deploy.yml`.
- `vite.config.ts` now uses `base: "/basic2dcad/"` for GitHub Pages.

Main functional features
- Drawing tools: select, line, potential, circle, arc, text, pin and pan.
- Selection supports move, resize, rotation, mirroring, alignment, group and ungroup.
- Multi-page projects with page tabs.
- Layers with visibility, locking, rename, active layer and move-to-layer workflows.
- Grid and snap settings are available in Settings.
- Fit-to-page exists in the toolbar and uses the same page framing logic as the initial auto-fit.

Potentials and pins
- Potentials can connect to pins while drawing.
- A nearby pin is highlighted and the potential end snaps to it.
- When a potential end is connected to a pin, the arrow and edge label on that side are hidden.
- Potentials can also connect to other potentials with T-connections and junction behavior.
- Same-number potentials now connect not only during initial drawing, but also after dragging a potential or dragging one of its endpoints onto another potential with the same number.
- Renumbering is available from the Potentials tab and follows visual order across pages.

Internationalization
- Added i18n with English and Brazilian Portuguese using `i18next` and `react-i18next`.
- Language selection is available in Settings and persisted in `localStorage`.
- Main visible UI is localized.
- Portuguese accent issues were fixed by using robust string encoding in translation resources.

Project format
- Project import/export now accepts only the current strict schema.
- `CadFile` uses only:
  - `version`
  - `layers`
  - `pages`
  - `components`
- Legacy or partial schemas are intentionally not supported anymore.

Components architecture
- The component system was refactored into two sources:
  - App Library
  - Project Components
- App Library components are loaded from JSON files in `src/library/components/*.json`.
- Project Components are stored inside the project JSON file.
- The built-in library is loaded through `src/library/index.ts`.
- Shared component helpers live in `src/utils/components.ts`.
- The components panel supports search and collapsible categories for the App Library.
- Redundant metadata text under component names was removed from the component cards to keep the UI cleaner.

Component workflow
- To create a reusable built-in component:
  1. draw the symbol on the canvas
  2. save it as a project component
  3. click `Download JSON`
  4. save the generated file
  5. later place that file in `src/library/components`
  6. rebuild and republish the app
- This replaced the earlier development-only write-to-disk export flow.
- The current approach works in production and is better for collaboration because contributors can design components in the running app and send JSON files.
- A bug where grouped components were inserted with a large offset was fixed. Grouped shapes no longer receive duplicated translation on placement.

Text, links and placeholders
- Text objects support optional internal links with `linkEnabled` and `linkTarget`.
- Navigation occurs with `Ctrl` or `Cmd` plus click.
- Special text placeholders currently supported:
  - `<project>`
  - `<drawing>`
  - `<author>`
  - `<page>`
  - `<n_pages>`
- These placeholders are resolved both on canvas and during PDF export.
